# ADR 0001: 选择 Slint 作为 GUI 框架

## 状态

已接受

## 背景

项目需要一个跨平台的桌面 GUI 框架来构建提醒应用的配置界面和提醒弹窗。

## 决策

选择 Slint 作为 GUI 框架。

## 考虑的选项

1. **Slint** - 声明式 UI，现代外观，但生态较小
2. **egui** - 跨平台、易学易用、活跃社区，但外观相对简单
3. **Tauri** - 使用系统 WebView，前端技术栈（React/Vue）
4. **Iced** - 受 Elm 启发，函数式风格
5. **Dioxus** - 类 React 的声明式 UI

## 决策理由

1. **现代外观**：Slint 提供原生外观和现代 UI 组件
2. **声明式 UI**：.slint 文件格式清晰，UI 与逻辑分离
3. **性能**：编译为原生代码，启动快、内存占用小
4. **适合工具类应用**：轻量级，适合桌面工具应用
5. **Windows 支持**：良好的 Windows 平台支持

## 影响

- 需要学习 Slint 的声明式语法
- 社区相对较小，遇到问题可能需要更多调试
- 后期如需跨平台，Slint 支持良好
