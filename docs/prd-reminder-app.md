# PRD: due - 桌面提醒应用

## Problem Statement

用户需要一个轻量级的桌面提醒工具，能够定时提醒、重复提醒，并支持时间区间设置。当前缺乏一个简单易用、常驻后台的提醒应用，能够在指定时间通过系统通知和窗口弹窗提醒用户。

## Solution

开发一个基于 Rust 的桌面提醒应用，使用 Slint 作为 GUI 框架，支持后台托盘运行。用户可以通过 GUI 界面创建和管理提醒，应用在指定时间通过系统原生通知和窗口弹窗双模式提醒用户。

## User Stories

1. As a 用户, I want a 创建定时提醒, so that 在指定时间收到一次性提醒
2. As a 用户, I want a 创建重复提醒, so that 按固定间隔重复收到提醒
3. As a 用户, I want a 设置重复提醒的时间区间, so that 只在指定日期范围内触发提醒
4. As a 用户, I want a 设置每日活跃时段, so that 只在每天的指定时间段内触发提醒
5. As a 用户, I want a 设置提醒标题, so that 快速了解提醒内容
6. As a 用户, I want a 设置详细提醒内容, so that 获取更详细的提醒信息
7. As a 用户, I want a 使用相对时间（如30分钟后）, so that 方便快捷地创建提醒
8. As a 用户, I want a 使用绝对时间（如2026-06-03 15:00）, so that 精确设置提醒时间
9. As a 用户, I want a 设置重复次数限制, so that 控制提醒的总触发次数
10. As a 用户, I want a 启用/禁用提醒, so that 临时暂停或恢复提醒
11. As a 用户, I want a 标记提醒为已完成, so that 跟踪提醒的处理状态
12. As a 用户, I want a 收到系统原生通知, so that 即使应用窗口最小化也能收到提醒
13. As a 用户, I want a 收到窗口弹窗提醒, so that 获得更醒目的提醒方式
14. As a 用户, I want a 关闭提醒弹窗, so that 确认收到提醒
15. As a 用户, I want a 延后提醒, so that 推迟一段时间后再收到提醒
16. As a 用户, I want a 选择延后时间（5/15/30/60分钟）, so that 灵活控制延后时长
17. As a 用户, I want a 标记提醒完成, so that 记录提醒已处理
18. As a 用户, I want a 应用启动后最小化到系统托盘, so that 不占用任务栏空间
19. As a 用户, I want a 双击托盘图标打开主窗口, so that 方便访问配置界面
20. As a 用户, I want a 右键托盘图标显示快捷菜单, so that 快速执行常用操作
21. As a 用户, I want a 查看提醒列表, so that 了解所有已创建的提醒
22. As a 用户, I want a 编辑提醒属性, so that 修改提醒的设置
23. As a 用户, I want a 删除提醒, so that 移除不再需要的提醒
24. As a 用户, I want a 数据存储在标准应用数据目录, so that 符合 Windows 应用规范
25. As a 用户, I want a 配置文件使用 TOML 格式, so that 方便手动编辑和备份

## Implementation Decisions

### 模块设计

1. **提醒模型模块（reminder）**
   - 定义 Reminder 结构体，包含标题、内容、时间设置、重复规则、状态等字段
   - 支持定时提醒和重复提醒两种类型
   - 重复提醒支持日期范围和每日活跃时段

2. **配置管理模块（config）**
   - 管理应用配置（如通知方式、默认延后时间等）
   - 使用 TOML 格式存储配置文件
   - 支持默认配置生成

3. **时间计算模块（time）**
   - 解析本地时间和相对时间（如"30分钟后"）
   - 计算重复提醒的下次触发时间
   - 处理时区和夏令时

4. **通知调度模块（scheduler）**
   - 管理提醒的调度和触发
   - 支持系统原生通知和窗口弹窗双模式
   - 处理延后和完成操作

5. **存储模块（storage）**
   - 提醒数据的持久化和加载
   - 使用 TOML 格式存储提醒数据
   - 存储位置：%APPDATA%\due\ 或 %LOCALAPPDATA%\due\

6. **GUI 模块（ui）**
   - 使用 Slint 构建界面
   - 左侧提醒列表 + 右侧详情编辑
   - 系统托盘集成

### 接口设计

- Reminder 模型提供创建、验证、序列化接口
- Config 模块提供读取、写入、默认配置接口
- 时间模块提供解析、计算、格式化接口
- 通知调度模块提供添加、移除、触发接口
- 存储模块提供加载、保存接口

### 技术选型

- GUI 框架：Slint
- 配置格式：TOML
- 序列化：serde + serde_derive
- 时间处理：chrono
- 系统通知：notify-rust
- 异步运行时：tokio

## Testing Decisions

### 测试原则

- 只测试外部行为，不测试实现细节
- 测试模块的公共接口
- 使用 mock 隔离外部依赖（如文件系统、系统通知）

### 测试模块

1. **Reminder 模型测试**
   - 测试提醒创建和验证
   - 测试序列化/反序列化
   - 测试重复提醒的时间区间设置

2. **Config 管理测试**
   - 测试配置文件读写
   - 测试默认配置生成
   - 测试配置验证

3. **时间计算测试**
   - 测试本地时间解析
   - 测试相对时间解析
   - 测试重复提醒触发时间计算
   - 测试时间区间判断

4. **通知调度测试**
   - 测试提醒调度逻辑
   - 测试触发判断
   - 测试延后和完成操作

5. **存储测试**
   - 测试提醒数据加载
   - 测试提醒数据保存
   - 测试文件格式兼容性

## Out of Scope

1. **开机自启动** - 后续版本实现
2. **标签分组** - 保持简单，暂不实现
3. **跨平台支持** - 当前仅支持 Windows
4. **声音提醒** - 暂不实现
5. **云端同步** - 暂不实现
6. **导入/导出** - 暂不实现

## Further Notes

- 应用使用后台托盘模式运行，不占用任务栏空间
- 配置文件和提醒数据存储在标准应用数据目录，符合 Windows 应用规范
- 使用 TOML 格式便于手动编辑和备份
- 优先实现核心功能，后续根据用户反馈迭代
